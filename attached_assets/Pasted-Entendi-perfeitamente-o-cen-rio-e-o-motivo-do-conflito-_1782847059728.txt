Se o sistema tenta encaixar a saída de 06:00 no dia seguinte como uma nova entrada, a folha de ponto vira um caos.A Solução: Fluxo Unificado de Via de Mão DuplaPara resolver isso, vamos ajustar a integração nos arquivos principais (control-id.ts, control-id-parsers.ts e nas rotas) para seguir estas regras estritas:1. Estrutura Rígida de 4 Batidas (Padrão Vigilante)O espelho de ponto passará a rastrear e organizar as batidas em 4 quadrantes fixos por jornada (seja ela diurna ou com virada de noite):P1: EntradaP2: Saída Almoço (Intervalo)P3: Retorno Almoço (Intervalo)P4: Saída2. Regra de Ouro do Write-Back (Sistema $\rightarrow$ Control iD)Quando você inserir uma batida manual no seu sistema:O sistema envia para a API do Control iD.O Ajuste: O sistema vai salvar o ID retornado pelo Control iD imediatamente na tabela control_id_punches. Isso "carimba" a batida. Quando o Cron rodar para buscar os dados do relógio, ele verá esse ID e saberá: "Essa batida eu já tenho, não preciso duplicar".3.
