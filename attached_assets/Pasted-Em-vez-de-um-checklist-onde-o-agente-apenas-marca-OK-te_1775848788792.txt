Em vez de um checklist onde o agente apenas marca "OK", teremos um Checklist com Prova de Vida e Validação de Ativos.

🧠 Como funcionará a "IA de Inspeção" (O Fluxo)
A ideia é que, ao subir a foto, o sistema não apenas guarde o arquivo, mas o envie para uma análise imediata de IA antes de permitir que o agente finalize o checklist.

Validação de Placa (OCR + Match): A IA lê a placa na foto e compara com o vehicle_id da OS. Se a foto for de um Fiat Uno e a OS for de uma Hilux, o sistema bloqueia na hora.

Reconhecimento de Objetos (De/Para): A IA identifica o formato do macaco, da chave de roda e até o estado dos pneus. Se o agente tirar foto do painel no lugar do macaco, a IA detecta a divergência.

Disparo de Alerta: Caso a IA encontre uma irregularidade (Placa errada, objeto ausente ou danificado), o sistema dispara o e-mail de alerta para o Thiago e para a Escolta.

🏗️ Estrutura Técnica para o Agent Implementar
Para que o Replit Agent coloque isso de pé sem erros, precisamos criar uma nova "camada" de inteligência no sistema.

1. Nova Tabela no Supabase: inspection_logs
Para registrar cada análise feita pela IA, guardando o que foi pedido, o que foi detectado e se houve divergência.

2. Integração de Visão (Gemini API ou Vision API):
O backend enviará a imagem para a IA com um prompt específico: "Valide se nesta imagem consta a placa ABC-123 e um macaco hidráulico em bom estado."

3. Gatilho de E-mail (Edge Functions):
Usaremos o Supabase Edge Functions para disparar o e-mail via Resend ou SendGrid assim que a divergência for detectada.
