O que está acontecendo é uma separação entre o Custo Fixo Estático e o Custo Operacional Variável (Real):

❌ Como está funcionando hoje:
O Balanço Gerencial que acabamos de ajustar puxa uma provisão fixa baseada no cadastro do funcionário (Aquele cálculo CLT padrão: Salário Base + 30% Periculosidade + Encargos fixos). Ele calcula o cenário "ideal" ou teórico do mês.

🎯 Como deve ser o funcionamento CORRETO (Integração com o Ponto Eletrônico):
Se o sistema já tem essa integração fantástica com o Control iD / iDFace Cloud que puxa as batidas a cada 5 minutos, o Balanço Gerencial deveria usar esses dados em tempo real para calcular a Folha Variável Real.

Para o cálculo bater 100% com a realidade do mês, o sistema precisa somar ao custo base do funcionário:

Horas Extras Reais: Calculadas a partir das batidas excedentes registradas no iDFace.

Adicional Noturno: Se o vigilante cumpriu jornada em horário noturno.

Faltas e Atrasos: Descontos reais cometidos no mês.

Sem essa conexão, se o funcionário fizer 20 horas extras na quinzena em missões de escolta, o custo dele continua aparecendo estático no Balanço, mascarando o gasto real com o RH.

Plaintext
Atue como Engenheiro de Software Sênior. O sistema já possui uma integração com o Ponto Eletrônico (Control iD / iDFace via Cloud) que busca batidas a cada 5 minutos. No entanto, o Balanço Gerencial ainda exibe apenas o custo fixo estatístico/provisionado do RH (baseado no cadastro).

Preciso unificar isso para que o Balanço Gerencial reflita o Custo Real da Folha, considerando o ponto eletrônico. Execute os seguintes passos de análise e implementação:

1. LOCALIZAR OS DADOS DO PONTO:
   - Identifique em qual tabela ou model do banco de dados estão sendo salvas as batidas, horas extras calculadas ou o espelho de ponto vindo do Control iD.

2. AJUSTAR A FUNÇÃO DE CÁLCULO DA FOLHA (`calcularFolha` / `rhSummary`):
   - Altere a lógica para que, no período selecionado no Balanço Gerencial, o sistema puxe o [Salário Base + Periculosidade + Encargos Fixos] e ADICIONE dinamicamente as Horas Extras e Adicionais Noturnos reais apurados no ponto eletrônico para cada agente.
