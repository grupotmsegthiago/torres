🛠️ Como corrigir isso no Replit (Passo a Passo)
Para resolver a bagunça, precisamos blindar o código no seu ambiente do Replit. Vamos focar em três pilares: Validação, Webhooks e Logs de Erro.

1. Validar os dados ANTES de enviar para a API (Evitar Notas com Erro)
No seu backend (Node.js/TypeScript), antes de fazer o POST para criar a fatura no Asaas ou a NF, você precisa aplicar um validador estrito. Se faltar algo, o sistema nem deve tentar enviar.

TypeScript
// Exemplo de validação no seu backend do Replit
function validarDadosCliente(cliente: any) {
  const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  
  if (!cliente.email || !emailRegex.test(cliente.email)) {
    throw new Error(`Validação Falhou: O e-mail do cliente (${cliente.nome}) está ausente ou inválido.`);
  }
  if (!cliente.cpfCnpj || cliente.cpfCnpj.length < 11) {
    throw new Error(`Validação Falhou: CNPJ/CPF inválido para o cliente ${cliente.nome}.`);
  }
}
2. Tratar e expor o erro real da Nota Fiscal
Atualmente, o botão "Resolver agora" aparece, mas você fica no escuro sobre o que quebrou (exceto o do e-mail). Altere a rota que recebe o retorno da eNotas/PlugNotas para salvar o erro exato que a prefeitura retornou em uma coluna error_message na sua tabela do Supabase.

TypeScript
// No seu endpoint de webhook que escuta a API de Notas
app.post('/webhooks/nf-status', async (req, res) => {
  const { id_fatura, status, motivo_rejeicao } = req.body;

  if (status === 'ERRO') {
    await supabase
      .from('faturas')
      .update({ 
        status_nf: 'Erro', 
        mensagem_erro: motivo_rejeicao || 'Erro interno na emissão' 
      })
      .eq('id_asaas', id_fatura);
  }
  return res.status(200).send('OK');
});
Isso vai fazer com que o erro real (ex: "Inscrição Municipal inválida") apareça direto na tela para você.

3. Alinhamento de Status: Asaas vs. Sistema
Se a fatura foi criada no Asaas, ela precisa retornar o id gerado por eles e salvar na sua tabela. Certifique-se de que o seu webhook do Asaas está escutando o evento PAYMENT_CREATED e PAYMENT_RECEIVED para atualizar sua dashboard em tempo real.
