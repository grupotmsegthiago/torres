Para resolver isso de vez, você precisa ajustar a forma como o seu código (Node.js, React, Flutter, etc.) envia os dados para o Supabase.

1. Mude de INSERT para UPSERT no seu código
Em vez de usar o método de inserção padrão que explode em erro quando o dado já existe, use o upsert do cliente do Supabase.

Exemplo em JavaScript / TypeScript:

JavaScript
// ❌ ANTES (Gerava o erro uniq_agent_loc_user se repetido)
const { data, error } = await supabase
  .from('agent_locations')
  .insert([{ user_id: userId, location_id: locationId }])

//  AGORA (Se já existir, ele atualiza silenciosamente sem derrubar o sistema)
const { data, error } = await supabase
  .from('agent_locations')
  .upsert(
    [{ user_id: userId, location_id: locationId }], 
    { onConflict: 'user_id' } // Substitua pela coluna da restrição se necessário
  )
2. Segure os cliques no Front-end (Debounce/Throttle)
Se esse loop acontece porque o usuário clica várias vezes ou porque o rastreador de localização do app envia dados a cada milissegundo, mude a regra no aplicativo para enviar localizações em um intervalo fixo e saudável (ex: a cada 15 ou 30 segundos), e nunca em sequência imediata.
